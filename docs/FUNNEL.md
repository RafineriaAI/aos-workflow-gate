# Self-Serve Activation and Commercialization Path

**Status: free self-serve advisory validation is open.** Guided onboarding,
paid pilots, Marketplace publication, efficacy claims, and a commercial offer
remain closed while the
[Hybrid Value Gate](../benchmarks/value/ASSESSMENT.md) is `NO_GO`.

The current path requires no mandatory contact and leaves the user with a
diagnosis and files that can be verified independently.

| Step | Surface | What the user leaves with | Evidence collected |
| --- | --- | --- | --- |
| Discover | Public repository, Pages, and exact-commit cases | A bounded claim and working quickstart | None by default |
| Adopt | Self-Test Mode, no checkout, read-only | A decision record from the user's own pipeline | Local evidence only |
| Understand | Job Summary: finding, effect, `Next` | One dominant action | Optional feedback |
| Verify | Offline `verify`, HTML evidence, trust docs | Independent confirmation of mechanics | User-controlled |
| Retain | Keep advisory for several weeks | Repeated exposure to real repository behavior | External study only |
| Escalate | Opens only after the required evidence gate | A bounded organization-level scope | Written acceptance criteria |

## First-value funnel

The activation target remains:

```text
install
-> exact-commit collection
-> one control-assurance diagnosis
-> one next action
-> replayable evidence
```

Time to first diagnosis, comprehension, and next-action clarity are activation
metrics. They are not proof of practical value unless the finding is accepted
or changes an action.

## Where a human is required

No RafineriaAI human is required to install or evaluate the free advisory
Action. Feedback is optional and opt-in. A human would enter only for a future
study or pilot scope after the applicable gate opens. The source remains
Apache-2.0, with no account, trial clock, or telemetry reporting that it was
evaluated.

## Commercialization gate

A paid path must not open merely because installation works. Advancement
requires external evidence for:

1. alert acceptance and actionable rate;
2. incremental findings beyond GitHub's own decision;
3. maintainer decision or remediation changes;
4. low repeated noise over several weeks;
5. retained installation and repository expansion;
6. a control owner willing to fund organization-level operation.

Stars, downloads, passing tests, internal red-team results, and case studies
without independent outcomes remain funnel observations, not demand evidence.

## Future paid hypothesis

If validated, the commercial layer would sell cross-repository control
inventory and drift, exception governance, evidence retention and export,
assurance reporting, and possibly managed signing. It would not sell generic
"protection quality," code review, or a promise that incidents cannot occur.
Policy packs alone are not treated as a sufficient paid moat.

Nothing in the free or future path makes a repository secure or compliant.
Decision records remain `UNSIGNED_NOT_OFFICIAL`.
