AOS Workflow Gate

GitHub can show green when no check is required, or when a PR changes the workflow that produced its own green checks.

AOS is a read-only pre-merge control check. It compares active branch requirements with checks on the exact commit, then returns PASSWARNBLOCK with one reason and one next action.

AOS Workflow Gate: WARN

What AOS found:
This PR changed a workflow that also produced checks used to assess
the same PR.

Effect: advisory only; this result does not fail the job.
Next: run or require one unchanged workflow check, then re-run AOS.

Not another AI reviewer. AOS does not judge code or guess who wrote it. Existing tools produce check results; AOS checks whether the repository merge controls actually ran and preserves the decision.
